نسخهٔ **v2.0.1** — یک patch release با چند بهبود رابط و سخت‌سازی exit-node. هیچ تغییر breaking ندارد و auto-updater آن را روی نصب‌های موجود v2.0.0 به‌صورت خودکار پیشنهاد می‌دهد.

• **تعویض حالت پروکسی به‌صورت زنده، بدون Stop/Start** ([1697e44](https://github.com/dazzling-no-more/rahgozar/commit/1697e44)). تغییر بین حالت‌های `direct` / `full` / `upstream` در رابط دسکتاپ حالا روی پروکسی فعال اعمال می‌شود بدون این‌که listenerها قطع شوند — دیگر چرخهٔ Stop → صبر → Start و افتادن اتصالات فعال در میانهٔ تعویض وجود ندارد.

• **لاگ‌های رنگی، ویرایشگر پارامترهای سفارشی، و endpoint جدید `/quota`** ([ee91d8e](https://github.com/dazzling-no-more/rahgozar/commit/ee91d8e)). لاگ‌های رابط حالا بر اساس سطح (info / warn / error) رنگ‌بندی شده‌اند تا اسکن چشمی سریع‌تر باشد، دیالوگ پارامترهای سفارشی به شما اجازه می‌دهد بدون ویرایش دستی `config.json` فیلدهای دلخواه را به payload ارسال‌شده به Apps Script اضافه کنید، و Apps Script یک endpoint `/quota` اضافه کرده که رابط می‌تواند با آن مصرف امروز کوتای رله را در برابر سقف روزانهٔ Google نمایش دهد.

• **پاسخ HTML تقلبی `200 OK` برای کاوش‌های ردشدهٔ exit-node** ([82797db](https://github.com/dazzling-no-more/rahgozar/commit/82797db)). وقتی exit-node یک PSK اشتباه، روش HTTP نادرست، یا JSON بدشکل دریافت می‌کند، حالا یک صفحهٔ HTML با وضعیت `200 OK` برمی‌گرداند به‌جای خطای 401/400. کاوش‌های فیلترگذاری که به دنبال پاسخ‌های متمایز خطا می‌گردند، یک صفحهٔ وب بی‌خطر می‌بینند نه یک fingerprint.

• **بومی‌سازی toggleهای Block QUIC / Block DoH / Bypass DoH در اندروید** ([0f1e336](https://github.com/dazzling-no-more/rahgozar/commit/0f1e336)). برچسب این سه toggle حالا ترجمهٔ فارسی در `values-fa/strings.xml` دارند و درست از resource خوانده می‌شوند (نسخه‌های hardcode شدهٔ پیشین در رابط دسکتاپ هم به‌عنوان عوارض جانبی حذف شدند).

• **اصلاح importهای exit_node — type-check تمیز بدون `@types/node`** ([2c7ec4b](https://github.com/dazzling-no-more/rahgozar/commit/2c7ec4b)). فایل‌های TypeScript مربوط به exit_node حالا با TypeScript استاندارد به‌درستی type-check می‌شوند: importهای سبک Deno با فرم‌های قابل‌حمل جایگزین شدند تا نیاز به `@types/node` از بین برود. تغییری کیفیتی، بدون اثر runtime.

**v2.0.1** is a patch release with a handful of UX polish wins and one exit-node hardening change. No breaking changes; the auto-updater will offer this to existing v2.0.0 installs automatically.

• **Hot-swap proxy mode without Stop/Start** ([1697e44](https://github.com/dazzling-no-more/rahgozar/commit/1697e44)). Switching between `direct` / `full` / `upstream` mode in the desktop UI now applies the new mode in-place while the proxy stays listening — no more Stop → wait → Start cycle that briefly dropped every active connection mid-switch.

• **Color-coded logs, custom-params editor, and `/quota` endpoint** ([ee91d8e](https://github.com/dazzling-no-more/rahgozar/commit/ee91d8e)). UI logs are now tinted by level (info / warn / error) for at-a-glance scanning, the custom-params dialog lets you add arbitrary fields to the Apps Script payload without hand-editing `config.json`, and Apps Script gained a `/quota` endpoint so the UI can surface today's relay-quota usage against the Google daily limit.

• **Decoy `200 OK` HTML response on rejected exit-node probes** ([82797db](https://github.com/dazzling-no-more/rahgozar/commit/82797db)). When the exit-node receives a wrong PSK, wrong HTTP method, or malformed JSON, it now returns a benign `200 OK` HTML page instead of a 401/400 error. Censorship probes that look for distinctive error responses get an innocent-looking webpage rather than a fingerprint.

• **Android Block QUIC / Block DoH / Bypass DoH toggles localized** ([0f1e336](https://github.com/dazzling-no-more/rahgozar/commit/0f1e336)). All three toggle labels now have Persian translations in `values-fa/strings.xml` and read from resources properly; the previously hardcoded desktop-UI copies of these strings were removed as a side effect.

• **exit_node imports cleaned up — typechecks under stock TypeScript** ([2c7ec4b](https://github.com/dazzling-no-more/rahgozar/commit/2c7ec4b)). Deno-style imports were swapped for portable forms so the exit_node TypeScript files now type-check cleanly without needing `@types/node`. Cosmetic for deployments that lint the script before deploying, no runtime effect.
