نسخهٔ **v2.4.0** — یک major release که در آن رابط کاربری دسکتاپ به‌طور کامل از egui به Tauri (بک‌اند Rust + فرانت‌اند Svelte/Tailwind) بازنویسی شده و حالا روی هر پلتفرم دسکتاپ یک installer بومی منتشر می‌کند. رفتار CLI و build اندروید عملاً تغییری نکرده است. به‌دلیل اینکه بستهٔ جدید دسکتاپ هویت باینری متفاوتی دارد و یک keypair updater تازه هم معرفی می‌کند، **کاربران فعلی نسخهٔ v2.3.x دسکتاپ باید یک‌بار به‌صورت دستی installer جدید را دانلود و نصب کنند** — updater داخلی v2.3.x این release را پیدا نمی‌کند. از v2.4.0 به بعد، Tauri updater مسئول آپدیت‌های in-app امضاشده می‌شود. کاربران CLI و اندروید همچنان از طریق مسیر امضای minisign موجود به‌طور خودکار آپدیت می‌گیرند.

• **بازنویسی کامل رابط کاربری دسکتاپ در Tauri (Svelte 5 + Tailwind v4)** ([87661d0](https://github.com/dazzling-no-more/rahgozar/commit/87661d0)). باینری egui به‌نام `rahgozar-ui` (حدود ۶۰۰۰ خط رابط immediate-mode Rust) جای خود را به یک app Tauri 2 می‌دهد: یک پل کوچک Rust در `desktop/src-tauri/src/commands.rs` کتابخانهٔ موجود `rahgozar` را از طریق IPC `invoke` در اختیار فرانت‌اند می‌گذارد، و فرانت‌اند یک درخت کامپوننت Svelte 5 + Tailwind v4 است. WebView توسط سیستم‌عامل ارائه می‌شود (WebView2 روی Windows، WebKit روی macOS/Linux)، بنابراین بستهٔ دسکتاپ دیگر wgpu/winit/glyphon را به‌عنوان dep مستقیم حمل نمی‌کند. رشته‌های دوزبانهٔ EN/FA با مدیریت صحیح RTL در `desktop/src/lib/i18n.svelte.ts` نگه‌داری می‌شوند، و دکمهٔ Copy و رفتار reveal-on-save پنل Recent log از v2.3.0 در `LogsTab.svelte` جدید حفظ شده‌اند.

• **عرضهٔ installerهای بومی برای هر پلتفرم، جایگزین launcher GUI درون آرشیو CLI** (matrix `desktop` در release.yml). نسخهٔ دسکتاپ حالا فایل‌های `.msi` (Windows)، `.dmg` و `.app.tar.gz` (macOS)، و `.deb` و `.AppImage` (Linux) را منتشر می‌کند. آرشیوهای headless `rahgozar-*.{tar.gz,zip}` همچنان برای سرورها، روترهای OpenWRT و هر کاربری غیرگرافیکی دیگر باقی می‌مانند — فقط دیگر launcher GUI با خود ندارند؛ این کار به‌عهدهٔ installerهای دسکتاپ است.

• **تأیید آپدیت‌های خودکار دسکتاپ از طریق امضای ed25519 پلاگین updater در Tauri**. هر installer دسکتاپ یک فایل `.sig` در کنار خود منتشر می‌کند؛ پلاگین `tauri-plugin-updater` پیش از اعمال هر آپدیت، امضا را در برابر public key که داخل app در حال اجرا embed شده تأیید می‌کند. keypair minisign که برای امضای CLI و APK استفاده می‌شود تغییری نکرده — فقط installerهای دسکتاپ از scheme جدید Tauri استفاده می‌کنند. manifest آپدیت (`latest.json`) برای هر release ساخته و به‌عنوان asset پیوست می‌شود؛ پلاگین به‌طور خودکار آدرس `releases/latest/download/latest.json` را poll می‌کند، بنابراین یک نصب v2.4.0 وقتی v2.5.0 منتشر شود آپدیت را به‌صورت in-app پیشنهاد می‌دهد.

• **نبود کد-امضای سطح سیستم‌عامل برای SmartScreen / Gatekeeper در installerهای فعلی**. کاربران Windows در اولین اجرا با هشدار "unrecognized app" در SmartScreen مواجه می‌شوند، و کاربران macOS باید با right-click → Open از Gatekeeper عبور کنند. تأیید امضای updater در Tauri که در بولت قبلی توضیح داده شد جداگانه از کد-امضای سطح OS است و همین حالا درست کار می‌کند — این مورد فقط تجربهٔ نصب اولیه را تحت‌تأثیر قرار می‌دهد. گواهی‌های EV و Apple Developer ID برای یک release بعدی برنامه‌ریزی شده‌اند.

• **به‌روزرسانی ابزار و محیط توسعه** ([c0dc7e6](https://github.com/dazzling-no-more/rahgozar/commit/c0dc7e6)، [0507e18](https://github.com/dazzling-no-more/rahgozar/commit/0507e18)، [177ee8d](https://github.com/dazzling-no-more/rahgozar/commit/177ee8d)). pinهای ابزار توسعه به‌روز شدند، `vite-env.d.ts` اضافه شد تا ابزار editor بتواند typeهای client مربوط به Vite را در پروژهٔ جدید Svelte resolve کند، و کامنت‌های stale مربوط به CI/MSRV همراه با اسکرین‌شات رابط بازنشسته‌شدهٔ egui حذف شدند.

**v2.4.0** is a major release: the desktop UI is fully rewritten from egui to Tauri (Rust backend + Svelte/Tailwind frontend), and ships native installers on every desktop platform. The CLI and Android APK builds are functionally unchanged. Because the new desktop bundle is a different binary identity AND introduces a fresh updater keypair, **existing v2.3.x desktop users will need to download the new installer manually one time** — the built-in updater on v2.3.x will not pick this release up. From v2.4.0 onward, the Tauri updater handles signed in-app upgrades automatically. CLI and Android continue updating normally via the existing minisign flow.

• **Desktop UI rewritten in Tauri (Svelte 5 + Tailwind v4)** ([87661d0](https://github.com/dazzling-no-more/rahgozar/commit/87661d0)). The egui `rahgozar-ui` binary (~6000 lines of immediate-mode Rust UI) is replaced by a Tauri 2 app: a small Rust bridge in `desktop/src-tauri/src/commands.rs` exposes the existing `rahgozar` library to the frontend via Tauri's `invoke` IPC, and the frontend is a Svelte 5 + Tailwind v4 component tree. The webview is system-provided (WebView2 on Windows, WebKit on macOS/Linux), so the desktop bundle no longer ships wgpu/winit/glyphon as direct deps. Bilingual EN/FA strings with proper RTL handling live in `desktop/src/lib/i18n.svelte.ts`, and the Copy button + reveal-on-save behaviour from v2.3.0's Recent log panel are preserved in the new `LogsTab.svelte`.

• **Native installers per platform, replacing the CLI-bundled GUI launcher** (release.yml `desktop` matrix). The desktop release now ships `.msi` (Windows), `.dmg` + `.app.tar.gz` (macOS), and `.deb` + `.AppImage` (Linux). The headless `rahgozar-*.{tar.gz,zip}` CLI archives still exist for servers, OpenWRT routers, and any other GUI-less use case — they just no longer carry a GUI launcher; the desktop installers do that.

• **Desktop auto-updates verified via Tauri's ed25519 updater signing**. Each desktop installer ships a `.sig` next to it; `tauri-plugin-updater` verifies the signature against a public key baked into the running app before applying any update. The minisign keypair used for CLI/APK signatures is unchanged — only desktop installers use the new Tauri scheme. The updater manifest (`latest.json`) is generated and attached to each release; the plugin polls `releases/latest/download/latest.json` automatically, so a v2.4.0 install will offer v2.5.0 in-app when the time comes.

• **Installers are not yet OS-level code-signed for SmartScreen / Gatekeeper**. First-launch Windows users will see the SmartScreen "unrecognized app" warning, and macOS users will need to right-click → Open to bypass Gatekeeper. The Tauri updater signature verification above is separate from OS-level code-signing and works correctly already — this only affects the first-install user-experience. EV / Apple Developer ID certs are planned for a later release.

• **Tooling / dev-environment refresh** ([c0dc7e6](https://github.com/dazzling-no-more/rahgozar/commit/c0dc7e6), [0507e18](https://github.com/dazzling-no-more/rahgozar/commit/0507e18), [177ee8d](https://github.com/dazzling-no-more/rahgozar/commit/177ee8d)). Bumped dev tooling pins, added `vite-env.d.ts` so editor tooling resolves the Vite client types in the new Svelte project, and removed stale CI/MSRV comments + a screenshot of the retired egui UI.
