نسخهٔ **v2.7.0** — یک minor release با تمرکز روی پوشش گسترده‌تر پلتفرم‌ها و کنترل دقیق‌تر روی شناسه‌های Deployment. حالا APK واحد روی Android TV هم نصب می‌شود، یک باینری portable برای ویندوز عرضه شده، Windows ARM64 (Snapdragon X / Surface Pro X+11) و چند هدف OpenWRT جدید (MIPSEL soft-float و armv7-musleabihf) به ماتریکس اضافه شده‌اند، و در رابط دسکتاپ و اندروید می‌توان هر شناسه را به‌صورت جداگانه فعال/غیرفعال کرد بدون اینکه از لیست حذف شود. هیچ تغییر breaking در API یا config وجود ندارد — نصب‌های فعلی CLI، دسکتاپ و اندروید در محل آپدیت می‌شوند.

• **تغییر وضعیت فعال/غیرفعال برای هر شناسهٔ Deployment به‌صورت ردیفی** ([02b1e0b](https://github.com/dazzling-no-more/rahgozar/commit/02b1e0b)). در رابط دسکتاپ و اندروید، کنار هر شناسه یک checkbox اضافه شده تا بتوانید بدون پاک کردن، آن را موقتاً از چرخش بیرون بکشید — مثلاً وقتی به سقف سهمیه نزدیک می‌شود یا برای A/B test یک deployment تازه. شناسه‌های غیرفعال در state ذخیره می‌مانند و کلاینت runtime فقط زیرمجموعهٔ `enabled: true` را round-robin می‌کند. فرمت `["A","B"]` قدیمی همچنان لود می‌شود (هر رشتهٔ خام به‌صورت `enabled: true` تفسیر می‌شود)، پس فایل‌های config دست‌نویس قبلی بدون migration کار می‌کنند.

• **پشتیبانی از Android TV، باینری portable ویندوز، و گسترش ماتریکس به Windows ARM64 و OpenWRT MIPSEL** ([d2376f4](https://github.com/dazzling-no-more/rahgozar/commit/d2376f4)). همان APK واحد به‌لطف اعلام leanback launcher روی Xiaomi MiBox، Nvidia Shield، Chromecast with Google TV و سایر باکس‌های Android TV قابل sideload شدن است (Play Store موضوع منتفی است چون این یک VPN-like proxy است). برای دسکتاپ، حالا یک `rahgozar-portable-windows-amd64.exe` کنار MSI/NSIS منتشر می‌شود — تک‌فایلی، بدون UAC و بدون نیاز به نصب، با signature minisign برای تایید دستی (به‌روزرسانی خودکار از داخل اپ portable باز هم به release page منتقل می‌شود چون `latest.json` فقط برای MSI ساخته شده است). ماتریکس CLI هم Windows ARM64 برای Copilot+ PCها (Snapdragon X / Surface Pro X+11)، MIPSEL soft-float برای روترهای MT7621، و armv7-musleabihf برای روترهای دههٔ ۲۰۱۰ OpenWRT (Archer C2600، Brume/Beryl، WRT3200ACM) را اضافه می‌کند. به‌علاوه، برای هر APK یک نسخهٔ per-ABI کوچک‌تر (~۱۵MB به‌جای universal ۵۰MB) منتشر می‌شود تا کاربران پشت تونل‌های سانسور باریک بتوانند فقط ABI دستگاه خود را دانلود کنند.

• **بهبود دقت auto-updater و رفع نام‌گذاری stale در نصب‌کننده‌های Tauri** ([d2376f4](https://github.com/dazzling-no-more/rahgozar/commit/d2376f4)). انتخاب asset به‌روزرسانی حالا بین glibc و musl و بین mips little و big endian به‌صورت سختگیرانه جدا می‌کند — یک باینری musl-arm دیگر به‌اشتباه `raspbian-armhf` glibc را دانلود نمی‌کند و در اولین dlopen کرش نمی‌دهد. هم‌چنین فایل `desktop/src-tauri/tauri.conf.json` در workflow `prepare-release` خودکار bump می‌شود، پس باگ release pageهای v2.5.x/v2.6.0 که MSI/AppImage/DMG با نام‌های `rahgozar_2.4.0_*` منتشر می‌کرد — چون این فایل از v2.4 دستی به‌روز نشده بود — حل شده است.

**v2.7.0** is a minor release focused on broader platform coverage and finer control over deployment IDs. The same APK now installs on Android TV, a portable Windows binary ships alongside the MSI, the matrix gains Windows ARM64 (Snapdragon X / Surface Pro X+11) plus a couple of new OpenWRT targets (MIPSEL soft-float and armv7-musleabihf), and the desktop and Android UIs gain a per-row enable/disable toggle so individual deployment IDs can be sidelined without removing them from the list. No breaking API or config changes — existing CLI, desktop, and Android installs upgrade in place.

• **Per-row enable/disable toggle for deployment IDs** ([02b1e0b](https://github.com/dazzling-no-more/rahgozar/commit/02b1e0b)). The desktop and Android UIs grow a checkbox next to each ID so you can temporarily pull one out of rotation without deleting it — handy when a deployment is approaching its daily quota or while you A/B a fresh one. Disabled IDs stay in the saved state and the runtime client only round-robins the `enabled: true` subset. The legacy `["A","B"]` form still loads (each bare string is interpreted as `enabled: true`), so existing hand-edited config files keep working without a migration.

• **Android TV support, Windows portable binary, and matrix expansion to Windows ARM64 + OpenWRT MIPSEL** ([d2376f4](https://github.com/dazzling-no-more/rahgozar/commit/d2376f4)). The same single APK can now be sideloaded on Xiaomi MiBox, Nvidia Shield, Chromecast with Google TV and other Android TV boxes thanks to a leanback launcher declaration (Play Store distribution is moot — rahgozar is a VPN-like proxy that wouldn't pass review anyway). On desktop, releases now ship `rahgozar-portable-windows-amd64.exe` next to the MSI/NSIS installers — single-file, no UAC, no install required, with a minisign signature for manual verification (in-app auto-update from the portable still redirects you to the release page, since `latest.json` only emits an MSI for Windows). The CLI matrix also adds Windows ARM64 for Copilot+ PCs (Snapdragon X / Surface Pro X+11), MIPSEL soft-float for MT7621 routers, and armv7-musleabihf for mid-2010s OpenWRT hardware (Archer C2600, Brume/Beryl, WRT3200ACM). On top of that, each release now publishes per-ABI Android APKs (~15 MB each, vs the ~50 MB universal) so users behind tight censorship tunnels can grab only the ABI their device needs.

• **Auto-updater asset picker tightened + stale Tauri bundle filenames fixed** ([d2376f4](https://github.com/dazzling-no-more/rahgozar/commit/d2376f4)). The update asset selection now strictly separates glibc vs musl and mips little vs big endian — a musl-arm binary will no longer accidentally download the glibc `raspbian-armhf` asset and crash on first dlopen. The `prepare-release` workflow also now auto-bumps `desktop/src-tauri/tauri.conf.json`, which fixes a bug where the v2.5.x and v2.6.0 release pages were shipping MSI/AppImage/DMG installers stamped `rahgozar_2.4.0_*` — that file had been bumped by hand for v2.4 and was never touched again until this release.
