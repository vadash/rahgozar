• **حالت Direct با قطعه‌بندی TLS برای دامنه‌های گوگل** ([913355d](https://github.com/dazzling-no-more/rahgozar/commit/913355d)). برای ترافیک گوگل، رهگذر دیگر از مسیر SNI-rewrite + MITM یا رلهٔ Apps Script استفاده نمی‌کند؛ به‌جای آن، ClientHello واقعی مرورگر را در چندین قطعهٔ TCP به edge گوگل می‌فرستد به‌طوری که DPI ایزی‌پی نتواند SNI را داخل یک پکت بازسازی کند. مرورگر هندشیک TLS واقعی با گوگل برقرار می‌کند — یعنی نیاز به نصب MITM CA برای دامنه‌های گوگل از بین می‌رود و سهمیهٔ Apps Script برای Gmail / Drive / Maps / YouTube مصرف نمی‌شود. در مواردی که هندشیک TLS با قطعه‌بندی کامل نشود، مسیر به‌طور شفاف به تونل SNI-rewrite قبلی fallback می‌کند، و یک circuit breaker پس از ۲ شکست متوالی برای ۶۰ ثانیه حالت Direct را غیرفعال می‌کند تا کاربران شبکه‌هایی که قطعه‌بندی روی آن‌ها کار نمی‌کند، در هر CONNECT گوگل هزینهٔ dial-then-race timeout را نپردازند. یک عارضهٔ جانبی مهم هم دارد: چون DPI دیگر SNI تمیز را نمی‌بیند، خطای Restricted Mode تحمیلی دولت ایران روی YouTube (پیام «این ویدئو محدود شده») دیگر برانگیخته نمی‌شود — کاربرانی که قبلاً مجبور بودند `youtube_via_relay` را برای دیدن ویدئوهای جدید روشن کنند، حالا با پیکربندی پیش‌فرض می‌توانند یوتیوب را به‌صورت عادی استفاده کنند. در حالت `apps_script`، دامنه‌های تحریم‌شدهٔ گوگل (Gemini، AI Studio، Bard، Labs) به‌جای مسیر مستقیم از طریق رلهٔ Apps Script هدایت می‌شوند چون گوگل IPهای ایرانی را برای این سرویس‌ها geo-block می‌کند. برای غیرفعال کردن کامل: `direct_mode.enabled: false` در `config.json`.

**v2.0.3** is a patch release adding TLS-fragmentation Direct Mode for Google-owned domains. Enabled by default; no breaking changes, and the auto-updater will offer this to existing v2.0.2 installs automatically.

• **TLS-fragmentation Direct Mode for Google domains** ([913355d](https://github.com/dazzling-no-more/rahgozar/commit/913355d)). For Google-served traffic, rahgozar no longer takes the SNI-rewrite + MITM path or the Apps Script relay; instead it forwards the browser's real TLS ClientHello to Google's edge directly, split across several TCP segments so DPI engines can't reassemble the SNI inside a single packet. The browser ends up with a real TLS session to the real origin — meaning no MITM CA install is needed for Google traffic, and no Apps Script quota is burned on Gmail / Drive / Maps / YouTube. When the fragmented TLS handshake doesn't complete, the path transparently falls back to the existing SNI-rewrite tunnel, and a circuit breaker engages after 2 consecutive total failures to skip Direct Mode for 60 s — so users on networks where fragmentation never works don't pay the dial-then-race latency on every Google CONNECT. There's a meaningful side-effect worth calling out: with ISP DPI no longer seeing a clean SNI, Iran's government-imposed YouTube Restricted Mode error (the "video restricted" page on new uploads) no longer fires — users who previously had to enable `youtube_via_relay` just to watch new content can now use YouTube normally with the default config. In `apps_script` mode, sanctioned Google products (Gemini, AI Studio, Bard, Labs) are routed through the Apps Script relay instead of the direct path because Google geo-blocks Iranian IPs for those services. To opt out entirely, set `direct_mode.enabled: false` in `config.json`.
