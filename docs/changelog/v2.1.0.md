نسخهٔ **v2.1.0** — یک minor release که شامل بهبود auto-update، بازنویسی قابل‌توجه مسیر رلهٔ Apps Script، تکمیل بومی‌سازی UI اندروید، و تشخیص bot-block روی مسیر exit-node است. auto-updater این نسخه را روی نصب‌های موجود v2.0.x به‌صورت خودکار پیشنهاد می‌دهد.

• **بررسی خودکار به‌روزرسانی هنگام اجرا با نشانگر دائمی** ([09c87f4](https://github.com/dazzling-no-more/rahgozar/commit/09c87f4)). هر دو برنامهٔ اندروید و UI دسکتاپ حالا هنگام راه‌اندازی به GitHub Releases مراجعه می‌کنند تا با نسخهٔ در حال اجرا مقایسه شوند. وقتی نسخهٔ جدیدتری موجود باشد، یک نشانگر non-blocking تا زمانی که کاربر آپدیت را نصب یا dismiss کند در UI باقی می‌ماند — پیش‌تر کاربر باید به یاد می‌آورد که پنل تنظیمات را باز کند و چک را به‌صورت دستی اجرا کند. این چک throttle می‌شود تا اجراهای پشت‌سرهم برنامه در یک بازهٔ کوتاه به GitHub API فشار نیاورند. unit test‌های جدیدی منطق مقایسهٔ نسخه و visibility نشانگر را پوشش می‌دهند.

• **بازنویسی مسیر رلهٔ Apps Script: classifier پاکت درخواست، حلقهٔ probe بلک‌لیست، و `apps_script_lang`** ([fb92374](https://github.com/dazzling-no-more/rahgozar/commit/fb92374)). بازنویسی قابل‌توجه مسیر رلهٔ Apps Script در سه بخش: یک classifier جدید envelope درخواست به‌ازای هر درخواست تصمیم می‌گیرد که از رله استفاده شود یا مسیر دیگر؛ یک حلقهٔ probe در پس‌زمینه هاست‌های blacklist‌شده را پیوسته بررسی می‌کند تا وقتی دوباره در دسترس شدند به‌طور خودکار از لیست خارج شوند؛ و گزینهٔ پیکربندی جدید `apps_script_lang` به کاربران اجازه می‌دهد Accept-Language رله را pin کنند تا سایت‌های مقصد UI خود را به زبان متناظر با locale حساب Google متعلق به deployment تغییر ندهند. این تغییرات روی هم استفادهٔ بی‌جا از رله را برای هاست‌هایی که مسیر جایگزین کاری دارند کاهش می‌دهد و از گیر کردن آیتم‌های blacklist جلوگیری می‌کند.

• **تشخیص bot-block روی مسیر exit-node با راهنمایی switch-host** ([35cf53c](https://github.com/dazzling-no-more/rahgozar/commit/35cf53c)). وقتی مسیر exit-node (Apps Script) در حال استفاده است، رهگذر حالا پاسخ‌ها را برای همان امضاهای bot-block CF/Akamai بررسی می‌کند که از قبل روی مسیر مستقیم تشخیص می‌داد. اگر IP خروجی رله flag شده باشد، کاربر یک پیام واضح «host را عوض کن» می‌بیند که به host list منتخب اشاره دارد، به‌جای یک خطای مبهم «blocked» که از down بودن خود مقصد قابل‌تمایز نبود.

• **همگام‌سازی گروه‌های GitHub fronting با patterniha v20** ([0d225f1](https://github.com/dazzling-no-more/rahgozar/commit/0d225f1)). گروه‌های domain-fronting مرتبط با GitHub در پیکربندی منتخب (api.github.com، raw.githubusercontent.com و غیره) به‌روز شدند تا با مجموعهٔ تأییدشدهٔ patterniha v20 مطابقت داشته باشند، و ترکیب‌های SNI کاری که از v2.0.0 چرخیده‌اند را بپوشانند.

• **تکمیل بومی‌سازی UI اندروید و پایان rebrand از mhrv به rahgozar** ([b7be244](https://github.com/dazzling-no-more/rahgozar/commit/b7be244)). رشته‌های انگلیسی hard-code شدهٔ باقی‌مانده در `HomeScreen.kt`، `AppPickerDialog.kt`، `ConfigSharing.kt`، `PipelineDebugOverlay.kt`، و `CaInstall.kt` به `strings.xml` / `values-fa/strings.xml` منتقل شدند، بنابراین دستگاه‌هایی با locale فارسی حالا یک UI کاملاً ترجمه‌شده می‌بینند. ارجاع‌های باقی‌مانده به «mhrv» در متن‌های user-facing (مستندات و label‌ها) به «rahgozar» تغییر کردند — package id داخلی `com.dazzlingnomore.mhrv` بدون تغییر می‌ماند تا تداوم به‌روزرسانی برای نصب‌های موجود حفظ شود.

• **تشخیص double-wrap بودن Code.gs پیش از v2.0.2 با خطای واضح** ([8f3b143](https://github.com/dazzling-no-more/rahgozar/commit/8f3b143)). اگر کاربر یک deployment exit-node را از یک setup پیش از v2.0.2 کپی کند که در آن `Code.gs` داخل یک تابع double-wrap شده بود، رله حالا این امضای ساختاری را شناسایی می‌کند و یک خطای قابل‌اقدام که به مستند setup فعلی اشاره دارد نمایش می‌دهد، به‌جای یک خطای generic JSON-parse که به کاربر در تشخیص مشکل کمکی نمی‌کرد.

**v2.1.0** is a minor release covering auto-update polish, a major rework of the Apps Script relay path, Android UI localization completion, and bot-block detection on the exit-node path. The auto-updater will offer this to existing v2.0.x installs automatically.

• **Auto-update check on launch with persistent indicator** ([09c87f4](https://github.com/dazzling-no-more/rahgozar/commit/09c87f4)). Both the Android app and the desktop UI now hit GitHub Releases at startup to compare against the running version. When a newer release is available, a non-blocking indicator stays visible in the UI until the user either installs the update or dismisses it — previously the user had to remember to open the settings panel and trigger the check manually. The check is throttled so repeated app launches in a short window don't hammer the GitHub API. New unit tests cover the version-comparison and indicator-visibility logic.

• **Apps Script relay rework: request-envelope classifier, blacklist probe loop, and `apps_script_lang`** ([fb92374](https://github.com/dazzling-no-more/rahgozar/commit/fb92374)). Substantial rework of the Apps Script relay path in three parts: a new request-envelope classifier decides per-request whether the relay should be used vs. another path; a background probe loop continuously rechecks blacklisted hosts so they can be cleared automatically when they come back online; and a new `apps_script_lang` config option lets users pin the relay's `Accept-Language` so destination sites don't switch UI to a language matching the deployment's Google-account locale. Together this reduces wasted relay calls on hosts that have a working alternative path and stops "stuck" blacklist entries.

• **Bot-block detection on the exit-node path with switch-host hint** ([35cf53c](https://github.com/dazzling-no-more/rahgozar/commit/35cf53c)). When the exit-node (Apps Script) path is in use, rahgozar now inspects responses for the same CF/Akamai bot-block signatures it already detects on the direct path. If the relay's egress IP has been flagged, the user sees a clear "switch the host" hint pointing at the curated host list, instead of an opaque "blocked" failure that looked identical to the destination simply being down.

• **GitHub fronting groups synced with patterniha v20** ([0d225f1](https://github.com/dazzling-no-more/rahgozar/commit/0d225f1)). The curated config's GitHub-related domain-fronting groups (api.github.com, raw.githubusercontent.com, etc.) are refreshed to match patterniha v20's vetted set, picking up working SNI combinations that have rotated since v2.0.0.

• **Android UI localization completed, mhrv→rahgozar rebrand finished** ([b7be244](https://github.com/dazzling-no-more/rahgozar/commit/b7be244)). The remaining hard-coded English strings across `HomeScreen.kt`, `AppPickerDialog.kt`, `ConfigSharing.kt`, `PipelineDebugOverlay.kt`, and `CaInstall.kt` have been extracted to `strings.xml` / `values-fa/strings.xml`, so Persian-locale devices now see a fully translated UI. The remaining "mhrv" references in user-facing text (docs and labels) are replaced with "rahgozar" — the internal package id `com.dazzlingnomore.mhrv` is unchanged to preserve update continuity for existing installs.

• **Pre-v2.0.2 Code.gs double-wrap detected with clear error** ([8f3b143](https://github.com/dazzling-no-more/rahgozar/commit/8f3b143)). If the user copies an exit-node deployment from a pre-v2.0.2 setup where `Code.gs` had been double-wrapped in a function, the relay now identifies the structural signature and surfaces an actionable error pointing to the current setup doc, instead of a generic JSON-parse failure that didn't help the user diagnose what was wrong.
