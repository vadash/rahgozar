نسخهٔ **v2.9.0** — یک minor release که تنظیمات بی‌ربط به مد فعال را پنهان می‌کند، دو crash در هنگام قطع اتصال اندروید را برطرف می‌کند، و stack اندروید را به Kotlin 2.1 و Compose 2025.04 ارتقاء می‌دهد.

• **پنهان کردن تنظیمات مد-بی‌ربط در دسکتاپ و اندروید** ([14d8729](https://github.com/dazzling-no-more/rahgozar/commit/14d8729)). پنل تنظیمات اکنون فقط گزینه‌هایی را نمایش می‌دهد که برای مد انتخاب‌شده (Apps Script، Drive، Full، Direct) معنادار هستند — مثلاً هنگام استفاده از Drive Mode، تنظیمات مخصوص Apps Script پنهان می‌شوند. هم در اندروید (HomeScreen.kt) و هم در دسکتاپ (TunnelTab.svelte) اعمال شده است.

• **ایزوله کردن VpnService در پروسهٔ جداگانهٔ `:vpn`** ([c7bd0af](https://github.com/dazzling-no-more/rahgozar/commit/c7bd0af)). سرویس VPN اکنون در یک پروسهٔ اندروید مجزا اجرا می‌شود تا یک crash تکراری در `libhwui` که هنگام بسته شدن VPN در کنار پروسهٔ UI رخ می‌داد را دور بزند. کلاس `VpnStateSync.kt` جدید هماهنگ‌سازی state بین پروسه‌ها را از طریق پیام‌رسانی `Binder` مدیریت می‌کند.

• **رد کردن `Tun2proxy.stop()` در Samsung اندروید ۱۶+** ([906a978](https://github.com/dazzling-no-more/rahgozar/commit/906a978)). در دستگاه‌های Samsung با اندروید ۱۶ به بالا، فراخوانی `Tun2proxy.stop()` هنگام disconnect یک FORTIFY abort در `libhwui` ایجاد می‌کرد. این fix آن فراخوانی را روی دستگاه‌های آسیب‌دیده رد می‌کند چون kernel پس از بسته شدن fd تونل خودش cleanup می‌کند.

• **ارتقاء stack اندروید: Kotlin 2.1 / Compose 2025.04** ([6add0a0](https://github.com/dazzling-no-more/rahgozar/commit/6add0a0)). کامپایلر و stdlib Kotlin به 2.1، BOM کتابخانه‌های Compose Material3 به 2025.04، و وابستگی‌های Rust-side به `jni` 0.22 و `rcgen` 0.14 ارتقاء یافتند.

• **ثبت وضعیت مسیریابی و skip‌های forwarder gate به‌ازای هر درخواست** ([164c504](https://github.com/dazzling-no-more/rahgozar/commit/164c504)). `proxy_server.rs` اکنون در سطح DEBUG ثبت می‌کند که آیا forwarder یک درخواست را gate کرده است و دلیل آن را نشان می‌دهد — برای دیباگ مسائل مسیریابی YouTube که در آن‌ها گاهی درخواست‌ها بدون خطا رها می‌شدند.

**v2.9.0** is a minor release that hides mode-irrelevant settings, fixes two Android VPN-disconnect crashes, and upgrades the Android stack to Kotlin 2.1 / Compose 2025.04.

• **Hide mode-irrelevant settings on desktop and Android** ([14d8729](https://github.com/dazzling-no-more/rahgozar/commit/14d8729)). The settings panel now shows only the options that apply to the active mode (Apps Script, Drive, Full, Direct) — for example, Apps Script-specific fields are hidden while Drive Mode is selected. Applied to both the Android `HomeScreen.kt` and the desktop `TunnelTab.svelte`.

• **Isolate VpnService in its own `:vpn` process** ([c7bd0af](https://github.com/dazzling-no-more/rahgozar/commit/c7bd0af)). The VPN service now runs in a dedicated Android process to dodge a recurring `libhwui` crash triggered when the VPN process shut down alongside the UI process. A new `VpnStateSync.kt` class handles cross-process state coordination via `Binder` messaging.

• **Skip `Tun2proxy.stop()` on Samsung Android 16+ disconnect** ([906a978](https://github.com/dazzling-no-more/rahgozar/commit/906a978)). On Samsung devices running Android 16+, calling `Tun2proxy.stop()` during disconnect caused a FORTIFY abort in `libhwui`. The fix skips that call on affected devices — the kernel cleans up after the tunnel fd is closed anyway.

• **Upgrade Android stack: Kotlin 2.1 / Compose 2025.04** ([6add0a0](https://github.com/dazzling-no-more/rahgozar/commit/6add0a0)). Kotlin compiler and stdlib bumped to 2.1, Compose Material3 BOM to 2025.04, and the Rust-side dependencies updated to `jni` 0.22 and `rcgen` 0.14.

• **Log routing state and per-request forwarder gate skips** ([164c504](https://github.com/dazzling-no-more/rahgozar/commit/164c504)). `proxy_server.rs` now logs at DEBUG level whether the forwarder is gating a request and why — improving observability for YouTube routing issues where requests were silently dropped without error.
