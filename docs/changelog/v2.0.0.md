اولین انتشار نسخهٔ جامعه‌محور **رهگذر** — ادامهٔ [mhrv-rs](https://github.com/therealaleph/MasterHttpRelayVPN-RUST) بعد از این که upstream برای مدتی غیرفعال شد و چندین PR ادغام‌نشده در صف ماند. v2.0.0 آن کارها را در یک حالت قابل نصب جمع می‌کند تا کاربران جایی برای دانلود build روزآمد داشته باشند.

نکات کلی:

• **این فورک کاملاً مستقل از upstream است.** Android applicationId جدید (`com.dazzlingnomore.mhrv`) به این معنی است که می‌توانی هر دو نسخه را کنار هم نصب کنی — رهگذر روی upstream نمی‌نویسد و برعکس.

• **کمک مالی به سه پروژهٔ بالادست برسد، نه به این فورک**: [@masterking32/MasterHttpRelayVPN](https://github.com/masterking32/MasterHttpRelayVPN) (پروژهٔ اصلی پایتون)، [@therealaleph/MasterHttpRelayVPN-RUST](https://github.com/therealaleph/MasterHttpRelayVPN-RUST) (پورت Rust با [sh1n.org/donate](https://sh1n.org/donate))، و [@patterniha/MITM-DomainFronting](https://github.com/patterniha/MITM-DomainFronting) (ایدهٔ گروه‌های CDN fronting). جزئیات بیشتر در [README](https://github.com/dazzling-no-more/rahgozar#credits).

## ویژگی‌های ادغام‌شده در v2.0.0

تمام موارد زیر از PRهای صف ادغام بالادست می‌آیند:

• **بهبود سرعت Apps Script — batch CacheService + DNS سمت edge** ([#958](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/958)). فراخوانی‌های `CacheService.get()` یکی‌یکی به فراخوانی‌های batch تبدیل شد و DNS hot-path روی edge بازنویسی شد. در عمل برای کاربرانی که چند درخواست هم‌زمان دارند کاهش latency ملموسی دارد.

• **YouTube — `relay_url_patterns` + SABR strip + exit-node-full SNI** ([#977](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/977)). راه‌های جدید برای رد کردن SafeSearch بر اساس SNI، گزینهٔ opt-in برای استریپ کردن SABR (برای رفع `Response too large` در ویدیوهای طولانی)، و سازگاری بهتر با exit-node-full.

• **بستهٔ آمادهٔ گروه‌های fronting CDN** ([#1033](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1033)). دکمهٔ **«Load curated fronting groups»** در رابط، یک‌ضرب گروه‌های Vercel، Fastly، AWS CloudFront، و مسیرهای مستقیم GitHub را نصب می‌کند. ساخته‌شده روی پروژهٔ [`patterniha/MITM-DomainFronting`](https://github.com/patterniha/MITM-DomainFronting).

• **ذخیره‌سازی چند-پروفایلی** ([#1057](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1057)). در دسکتاپ و اندروید می‌توانی چند پروفایل کانفیگ داشته باشی (مثلاً «خانه»، «کار»، «full tunnel») و بدون تایپ مجدد deployment ID و auth_key بین‌شان جابه‌جا شوی.

• **استفاده از رهگذر به‌عنوان upstream proxy برای Psiphon / xray** ([#1189](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1189)). در حالت `direct`، رهگذر یک پروکسی HTTP/SOCKS5 ارائه می‌دهد که سایفون یا xray می‌توانند به‌عنوان upstream proxy خود استفاده کنند — هاست‌های fronted نشده به‌صورت raw TCP عبور می‌کنند تا ترافیک bootstrap سایفون به سرورهای خودش برسد.

• **به‌روزرسان خودکار درون-برنامه** ([#540](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/540)). دسکتاپ و اندروید هر دو حالا بررسی به‌روزرسانی، دانلود، و نصب release جدید را در داخل برنامه انجام می‌دهند.

• **گسترش گروه‌های Vercel/Fastly + اضافه شدن github-central/alive/gist/pubmed** ([#1191](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1191) از @Shjpr9، cherry-pick از بالادست). IP/SNI تازه برای Vercel و Fastly، تغییر نام `netlify` → `amazon-cloudfront`، و ۴ گروه fronting جدید برای GitHub Copilot / live / gist و PubMed Central.

## Features integrated into v2.0.0

All of the following come from upstream's queued-PR backlog:

• **Apps Script perf wins — batch `CacheService.getAll` + edge-DNS hot-path** ([#958](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/958)). N+1 `CacheService.get()` calls collapsed into batched `getAll()` calls, and the DNS hot-path on the Apps Script edge was rewritten. Material latency drop for users running multiple concurrent requests.

• **YouTube — `relay_url_patterns` + SABR strip + exit-node-full SNI** ([#977](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/977)). New `relay_url_patterns` config field for path-pinned routing through the Apps Script relay, opt-in `sabr_strip` for fixing `Response too large` errors on long-form videos, and better compatibility with the exit-node-full deployment.

• **Bundled curated CDN fronting groups + UI loaders** ([#1033](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1033)). New **"Load curated fronting groups"** button in the UI one-taps Vercel, Fastly, AWS CloudFront, and direct GitHub routes. Built on top of [`patterniha/MITM-DomainFronting`](https://github.com/patterniha/MITM-DomainFronting)'s research.

• **Multi-profile config storage** ([#1057](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1057)). Desktop and Android both gain named config profiles (think "Home", "Work", "Full tunnel"), so you can switch between deployment IDs / auth keys without retyping. profiles.json mirrors the desktop UI's profile store.

• **Use rahgozar as an upstream proxy for Psiphon / xray** ([#1189](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1189)). In `direct` mode, rahgozar exposes its HTTP / SOCKS5 listeners as proxies that Psiphon or xray can point at — un-fronted hosts pass through as raw TCP so Psiphon's bootstrap traffic reaches its own servers untouched. See [docs/use-as-upstream.md](https://github.com/dazzling-no-more/rahgozar/blob/main/docs/use-as-upstream.md).

• **In-app auto-updater** ([#540](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/540)). Desktop and Android can now check for updates, download the matching release asset, verify it, and stage the swap for next launch. Optional minisign verification on top (off by default in this fork — disabled until release signing keys are minted).

• **Vercel/Fastly refresh + github-central/alive/gist/pubmed groups** ([#1191](https://github.com/therealaleph/MasterHttpRelayVPN-RUST/pull/1191) by @Shjpr9, cherry-picked from upstream). Fresher IP/SNI tuples for Vercel and Fastly, `netlify` group renamed to `amazon-cloudfront` (more accurate — the edge is CloudFront), and 4 new groups for GitHub Copilot / live / gist and PubMed Central.

## Compatibility notes

• **Auto-updater enforces minisign signature verification.** Every release asset is signed by the workflow with a key held only in GitHub Actions Secrets; the corresponding pubkey is embedded into the binary at compile time. The auto-updater refuses to install any future update whose `<asset>.minisig` doesn't verify against the embedded pubkey — so a compromised Releases page alone can't push a malicious binary, only a compromise of the GitHub Actions secret store. First-install users grab the artifact from the Release page directly (no signature check needed for that path); the verification kicks in on subsequent in-app updates.

• **Side-by-side install with upstream is supported.** Different Android applicationId, different desktop data dir, different macOS bundle identifier (`com.dazzlingnomore.rahgozar`). Configs are not auto-migrated; copy your `config.json` from the upstream data dir to the rahgozar data dir if you want to bring settings across.

• **First-release platform coverage** matches upstream v1.9.25 (Linux amd64/arm64/armhf/musl-amd64/musl-arm64, OpenWRT mipsel-softfloat, macOS amd64/arm64, Windows amd64, Android universal + 4 per-ABI APKs, Docker tunnel-node at `ghcr.io/dazzling-no-more/rahgozar-tunnel-node:2.0.0`). All linux cross-compile is now via github-hosted runners + apt-installed gcc-cross / `messense/rust-musl-cross` docker images instead of upstream's self-hosted Hetzner runner fleet.
