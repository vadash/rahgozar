**v2.5.1** is a patch release that publishes `tunnel-node` as a static binary alongside the existing Docker image. Added at user request — running the full-tunnel bridge on a small VPS no longer requires Docker; download the tarball, point a systemd unit at it, and you're done. No changes to the CLI, desktop, Android, or tunnel protocol.

• **Static `tunnel-node` binary release** ([07f203f](https://github.com/dazzling-no-more/rahgozar/commit/07f203f)). Two new archives land on the release page: `rahgozar-tunnel-node-linux-musl-amd64.tar.gz` and `rahgozar-tunnel-node-linux-musl-arm64.tar.gz`. Both are fully-static musl builds — they run on any Linux distro (Alpine, Debian, Ubuntu, RHEL, …) without libc compatibility concerns. A complete walkthrough (with a sample systemd unit) is in [tunnel-node/README.md](../../tunnel-node/README.md). The Docker image (`ghcr.io/dazzling-no-more/rahgozar-tunnel-node`) keeps publishing — this is an additional option, not a replacement.
