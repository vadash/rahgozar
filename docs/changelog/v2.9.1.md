• **نام بردن از هر دو هدف بازنشر در خطای double-wrap مد CFW** ([44bce6c](https://github.com/dazzling-no-more/rahgozar/commit/44bce6c)). این خطا زمانی رخ می‌داد که حداقل یک لایه در مد CFW (Apps Script یا Cloudflare Worker) همچنان کد pre-v2.0.2 را اجرا می‌کرد، اما متن قبلی فقط Code.gs را نام می‌برد و به worker.js تنها به‌عنوان یک نکتهٔ فرعی اشاره می‌کرد — به‌راحتی از قلم می‌افتاد. پیام جدید هر دو هدف بازنشر را به‌طور یکسان فهرست می‌کند و توضیح می‌دهد که paste کردن کد به معنای deploy نیست (Apps Script نیازمند «Manage deployments → New version» است؛ Cloudflare نیازمند «Save and deploy» است، نه فقط «Save»).

**v2.9.1** is a patch release that fixes a DataCloneError crash when opening the SNI Pool modal and makes the exit-node double-wrap error point at the right redeploy target.

• **Fix SNI Pool modal crashing with DataCloneError** ([44bce6c](https://github.com/dazzling-no-more/rahgozar/commit/44bce6c)). Clicking the "SNI pool" button on the Tunnel tab threw `DataCloneError: Failed to execute 'structuredClone' on 'Window': [object Array] could not be cloned` and the list never rendered. Svelte 5's reactive `$state` proxies aren't compatible with `structuredClone`; the modal now clones the plain array returned by the API before assigning it to state. Same latent bug fixed preemptively in the TunnelTab Revert path.

• **Name both redeploy targets in CFW double-wrap error** ([44bce6c](https://github.com/dazzling-no-more/rahgozar/commit/44bce6c)). The error fired when at least one CFW-mode layer (Apps Script or the Cloudflare Worker) was still running pre-v2.0.2 code, but the previous message named only Code.gs and mentioned worker.js as an afterthought — easy to miss. The new message lists both redeploy targets equally and spells out that pasting code is not deploying it (Apps Script needs "Manage deployments → New version"; Cloudflare needs "Save and deploy", not just "Save").
